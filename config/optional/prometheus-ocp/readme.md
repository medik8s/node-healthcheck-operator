# Platform Prometheus Metrics Configuration

This configuration enables metrics collection via OpenShift Platform Prometheus (prometheus-k8s in openshift-monitoring namespace) using TLS client certificate authentication (mTLS).

## Requirements

- OpenShift Container Platform 4.14+
- The `scrapeClass: tls-client-certificate-auth` feature requires OCP 4.14 or later

## How It Works

The Node Health Check Operator serves metrics directly with mTLS using `k8s.io/apiserver/pkg/server/dynamiccertificates`:

1. **Serve TLS**: Using a certificate generated by OpenShift's service-serving-cert-signer (mounted at `/etc/tls/private/`)
2. **Verify Client Certificates**: Using the CA from `kube-system/extension-apiserver-authentication`, read directly via an in-process informer (no copy ConfigMap needed)
3. **mTLS Authentication**: Go's TLS stack verifies that Prometheus presents a client certificate signed by the cluster CA. Only holders of valid client certificates can access `/metrics`

There is no kube-rbac-proxy sidecar — the manager handles TLS and client certificate verification directly.

## Components

### ServiceMonitor (`monitor.yaml`)

Configures Platform Prometheus to scrape metrics from the operator:
- Uses `scrapeClass: tls-client-certificate-auth` for client certificate authentication
- No bearer token required (unlike User Workload Prometheus)

### CA Bundle ConfigMap (`ca-configmap.yaml`)

A ConfigMap with the `service.beta.openshift.io/inject-cabundle: "true"` annotation. OpenShift's service-ca-operator automatically populates it with the cluster's service CA certificate. This CA is used by Prometheus to verify the metrics endpoint's server certificate.

### Prometheus RBAC (`prometheus-k8s-role.yaml`, `prometheus-k8s-rolebinding.yaml`)

Namespace-scoped Role and RoleBinding that allow the `prometheus-k8s` service account to discover services, endpoints, pods, and endpointslices in the operator namespace.

### kube-system RBAC (`../kube-system-rbac/`)

Role and RoleBinding in the `kube-system` namespace that allow the operator to read the `extension-apiserver-authentication` ConfigMap containing the client CA certificate.

## Automatic Setup

When installing the operator via OLM, the namespace will be created with the `openshift.io/cluster-monitoring: "true"` label, which allows Platform Prometheus to discover ServiceMonitors in this namespace.

All required resources (ConfigMaps, RBAC) are automatically created as part of the bundle.

## Manual Setup (Development)

If deploying manually without OLM, ensure the namespace has the cluster-monitoring label:

```bash
kubectl label namespace openshift-workload-availability openshift.io/cluster-monitoring=true --overwrite
```

## Verifying Metrics Collection

1. Check that the ServiceMonitor exists:
   ```bash
   kubectl get servicemonitor -n openshift-workload-availability
   ```

2. Check Platform Prometheus targets:
   ```bash
   oc exec -n openshift-monitoring prometheus-k8s-0 -c prometheus -- \
     curl -s 'http://localhost:9090/api/v1/targets' | \
     jq '.data.activeTargets[] | select(.labels.job | contains("node-healthcheck"))'
   ```

3. Query metrics:
   ```bash
   oc exec -n openshift-monitoring prometheus-k8s-0 -c prometheus -- \
     curl -s 'http://localhost:9090/api/v1/query?query=up{job=~".*node-healthcheck.*"}' | jq
   ```

4. Run the verification script:
   ```bash
   ./hack/verify-platform-metrics.sh
   ```

## Troubleshooting

### Target shows DOWN in Prometheus

Check manager logs for TLS errors:
```bash
kubectl logs -n openshift-workload-availability deployment/node-healthcheck-controller-manager -c manager | grep -i "cert\|tls\|mTLS"
```

Verify the TLS secret exists (created by service-serving-cert-signer):
```bash
kubectl get secret node-healthcheck-tls -n openshift-workload-availability
```

Verify the kube-system RBAC allows reading the client CA:
```bash
kubectl get role node-healthcheck-manager-role -n kube-system
kubectl get configmap extension-apiserver-authentication -n kube-system -o jsonpath='{.data.client-ca-file}' | head -1
```

### ServiceMonitor not discovered

Verify the namespace label:
```bash
kubectl get ns openshift-workload-availability --show-labels | grep cluster-monitoring
```
